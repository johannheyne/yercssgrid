about

I personel use this Grid to get things done on my clients projects. But it is still an evolution of understanding how you manage the complexity of responsive-designed websites with basic css features. There will be bether solutions in future.

You can define the number of columns, different gap-widths and breakpoints. All together this parameters are combined in the following principle: .{classtype}-{breackpoint}-{value}. There are some more options you can find in the settings and at this demo page.

conventions

Because of the technic to layout the gaps with padding and negative margins, we need in some cases overflow hidden on the parent element to prevent the negativ horizontal margins of the grid protruding about the body and causes horizontal scrolling. And we need border-box as box-sizing on all elements.

<body>
.grid .gap-0-40
.col-0-6
content
.col-0-6
content
.grid_wrap { overflow: hidden; }
* { box-sizing: border-box; }

setup

In the file /sass/yercssgrid.scss we have some variables to setup the grid.

$col-default-font-size

The columns of the grid are generated by setting the columns elements property display to inline-block. This causes unwanted spaces between the columns which can be prevented by setting the property font-size of the parent element to 0. Therefore, the font size must be adjusted again within the column elements.

$col-default-font-size: 16px;

$grid-use-cols $grid-use-perc

The grid is made of columns with percentage width values. Therefore it is possible to use grid-cols and/or strait percentage values to define the width of a col. These variables define, what should be possible.

$grid-use-cols: yes;
$grid-use-perc: no;

$grid-cols

Defined the number of grid columns.

$grid-cols: 12;

All variables:

$grid-use-breackpoint-0: yes;

$col-default-font-size: 16px;

$grid-use-breackpoints: yes;
$grid-breackpoint-step: 100;
$grid-breackpoint-min: 300;
$grid-breackpoint-max: 1000;

$grid-use-gaps: yes;
$grid-gap-step: 10;
$grid-gap-max: 20;

$grid-use-cols: yes;
$grid-use-cols-push: yes;
$grid-use-cols-pull: yes;
$grid-cols: 12;

$grid-use-perc: no;
$grid-use-perc-push: no;
$grid-use-perc-pull: no;
$grid-perc-step: 10;

$grid-use-text-align-inherit: yes;

$grid-use-top-bottom: yes;
$grid-top-bottom-step: 10;
$grid-top-bottom-min: 0;
$grid-top-bottom-max: 30;

$grid-use-display: yes;

browser support

In the latest Safari for Mac, its rendering engine has a little trouble with long decimal places like 33,33333333%. This means that sometimes the rightmost column is of one pixel distance to the right edge. There is still nothing easy I can do to fix this without stupid conventions.

Version 2 2014.02.20

This changes may need your attention on how to include the yercssgrid files.

  1. The setup file yercssgrid-setup.scss is no longer imported into the main yercssgrid.scss file. The yercssgrid.scss code is outsourced in yercssgrid-generator.scss. You now can import your own setup file before the import of the generator file.

  1. Added breakpointbased hide and show support

Version 1 2014.02.01

  1. Added SASS support

Version 0.2 2013.11.25

  1. In version 0.1, the cols were layouted with float. Here, however, there were problems with the layout of cols over several rows with different cols-heights. Therefore, the cols are now layouted using inline-block.
