YerCSSGrid

The main structur of classe are {classtype}-{breackpoint}-{value}.

conventions

Because of the technic to layout the gaps with padding and negative margins, we need overflow hidden on the body and overflow auto on the html element. Otherwise the grid element with negativ horizontal margins could protrude about the body and causes horizontal scrolling. And we need border-box as box-sizing on all elements.

<body>
.grid .gap-0-40
.col-0-6
content
.col-0-6
content
html { overflow: auto; }
body { overflow: hidden; }
* { box-sizing: border-box; }

pushing columns

We center a single column depending on two breakpoints ( 800, 1000 ) Each breakpoint has a different column width and push behavior.

<div class="grid">
  <div class="col-800-10 push-800-1 col-1000-8 push-1000-2"></div>
</div>
col-800-10 push-800-1 col-1000-8 push-1000-2

text-align inherit

On very small screens it is sometimes usefull to have centered content. With "text-align-inherit-{breakpoint}" we can get the centered text-alignment of by setting text-align to inherit on a breakpoint.

<div class="grid gap-0-20">
  <div style="text-align: center;" class="col-600-6 text-align-inherit-600">1</div>
  <div style="text-align: center;" class="col-600-6 text-align-inherit-600">2</div>
</div>

setup

In the file yercssgrid.less we have some variables to setup the grid.

@grid-use-cols @grid-use-perc

The grid is made of columns with percentage width values. Therefore it is possible to use grid-cols and/or strait percentage values to define the width of a col. These variables define, what should be possible.

@grid-use-cols: yes;
@grid-use-perc: no;

@grid-cols

Defined the number of grid columns.

@grid-cols: 12;

browser support

In the latest Safari for Mac, its rendering engine has a little trouble with long decimal places like 33,33333333%. This means that sometimes the rightmost column is of one pixel distance to the right edge. There is still nothing easy I can do to fix this without stupid conventions.
